Look at the files src/reconciliation/likelihood.py and src/reconciliation/reconcile.py
Inside, I put functions E_step and Pi_step. These functions are contracting functions of the tensors E and Pi respectively. Therefore, the iterates converge to a fixed point. The converged tensor Pi then represents some probabilities for a reconciliation, as follows from the ALEml_undated algorithm. However, both contracting functions depend on a parameter theta. This parameter theta needs to be optimized so as to maximize the log-likelihood. We can do it by gradient descent on the negative log-likelihood, but we can't just differentiate Pi_step, because that doesn't correspond to differentiating the fixed point. What we need to do instead is work with vector jacobian products to obtain the implicit gradient. This process is described in the file instructions.md. I want you to read all three files, then create a file src/optimization/implicit_grad.py, in which you will implement the functions necessary to obtaining the implicit gradient. You will test your implementation on the datasets in test_trees_1, test_trees_2, test_trees_3: to test it, you can compare the implicit differential to the differential obtained with small variations of theta. You can open the datasets using the function in reconcile.py. The final test would be to perform gradient descent and obtain a theta parameter that matches closely the parameters in files like /home/enzo/Documents/git/WP2/gpurec/tests/data/test_trees_1/output/model_parameters/model_parameters.txt and the likelihood in files like gpurec/tests/data/test_trees_1/output/per_fam_likelihoods.txt.
